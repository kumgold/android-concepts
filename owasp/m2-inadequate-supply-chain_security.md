# M2: Inadequate Supply Chain Security (부적절한 공급망 보안)

## 개요
공격자는 모바일 앱 공급망의 취약점을 악용하여 애플리케이션 기능을 조작할 수 있다. 예를 들어, 모바일 앱 코드베이스에 악성 코드를 삽입하거나
빌드 과정에서 코드를 수정하여 백도어, 스파이웨어와 같은 악성 코드를 심을 수 있다.

- 악용 : 보통
- 탐지 : 어려움
- 기술적/비즈니스 영향 : 심각

## 위험 환경
- 서드파티 컴포넌트의 보안 부족 : 라이브러리나 프레임워크와 같은 서드파티 컴포넌트에 공격자가 악용할 수 있는 취약점이 포함될 수 있다.
모바일 애플리케이션 개발자가 서드파티 컴포넌트를 제대로 검증하지 않거나 업데이트 하지 않으면 공격에 취약할 수 있다.
- 악의적인 내부적 위협 : 악의적인 내부 개발자나 공급업체가 의도적으로 애플리케이션에 취약점을 주입할 수 있다. 이는 프로세스에 대한 적절한 보안 통제 및 모니터링을 구현하지 않을 경우
위험할 수 있다.
- 부적절한 테스트 및 검증 : 모바일 애플리케이션 개발자가 애플리케이션을 철저히 테스트 하지 않으면 공격에 취약할 수 있다. 또한 공급망 프로세스의 보안을 검증하는 것도 중요하다.
- 보안 인식 부족 : 모바일 애플리케이션 개발자가 적절한 보안 인식을 갖고 있지 않으면 공급망 공격을 방지하는 데 필요한 보안 통제를 구현하지 않을 수 있다.

## 시나리오
- 시나리오 1(악성코드 주입) : 공격자가 개발 단계에서 악성 코드를 주입한다. 공격자는 유효한 인증서로 앱에 서명하고 앱 스토어 보안 검사를 우회하여 배포한다.
사용자는 감염된 앱을 다운로드하고 설치하고 공격자는 사용자의 민감한 개인 데이터를 훔쳐 금전적인 피해를 입힐 수 있으며, 앱 제공업체의 평판에 손상을 입힐 수 있다.

## 방지
- 모바일 앱 개발 수명 주기 전반에 걸쳐 보안 코딩 관행, 코드 검토 및 테스트, 모의 해킹 등을 구현하여 취약점을 식별하고 보완한다.
- 공격자가 악성 코드를 서명하고 배포하는 것을 방지하기 위해 안전한 앱 서명 및 배포 프로세스, 모니터링을 강화한다.
- 취약점의 위험을 줄이기 위해 신뢰할 수 있고 검증된 서드파티 라이브러리를 사용한다.
- 공격자가 앱의 취약점을 악용하는 것을 방지하기 위해 앱 업데이트, 패치 및 릴리스에 대한 보안 통제를 수립한다.
- 보안 테스트, 스캐닝과 같은 기술을 통해 공급망 보안 사고를 모니터링 하고 탐지하여 사고에 시기적절하게 대응한다.
