# M1: Improper Credential Usage (부적절한 자격 증명 사용)

## 개요
모바일 애플리케이션에서 하드코딩된 Credential(자격 증명) 혹은 부적절한 Credential을 사용할 경우 앱은 위협에 노출된다.

- 악용 : 쉬움
- 탐지 : 쉬움
- 기술적/비즈니스 영향 : 심각

## 위험 환경
- 하드코딩된 Credential : 모바일 앱의 소스 코드나 파일 내에 하드코딩된 Credential이 포함되어 있다면 앱은 공격에 취약하다.
- 안전하지 않은 Credential 전송 : 암호화 없이 혹은 안전하지 않은 채널을 통해 전송한다면 앱은 공격에 취약하다.
- 안전하지 않은 Credential 저장 : 모바일 앱이 사용자 Credential을 안전하지 않은 방식으로 저장하는 경우 앱은 공격에 취약하다.
- 취약한 사용자 인증 : 사용자 인증이 취약한 프로토콜에 의존하거나 쉽게 우회할 수 있다면 앱은 위험에 노출될 수 있다.

## 공격 시나리오 예시
- 시나리오 1(하드코딩된 Credential) : 공격자가 모바일 앱 소스 코드 내에서 하드코딩된 Credential을 탈취한다. Credential을 사용하여 앱 또는 백엔드 시스템 내부에 민감한 기능에 대한 무단 접근 권한을 얻을 수 있다.
- 시나리오 2(안전하지 않은 Credential 전송) : 공격자가 모바일 앱과 백엔드 시스템 사이에 안전하게 전송하지 않는 Credential을 가로챈다. 마찬가지로 무단 접근 권한을 얻을 수 있다.
- 시나리오 3(안전하지 않은 Credential 저장) : 공격자가 기기에 물리적으로 접근하여 앱에 저장된 Credential을 추출한다. 마찬가지로 무단 접근 권한을 얻을 수 있다.

## 방지
- Credential 전송 중 암호화한다.
- 사용자 기기에 저장하지 않는다. 대신 안전하고 취소가 가능한 액세스 토큰을 사용한다.
- 사용되는 모든 API Key 혹은 토큰을 정기적으로 업데이트하고 교체한다.
- 강력한 사용자 인증 프로토콜 구현
